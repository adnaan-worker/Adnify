# Agent Settings UI Readability Design

**背景**

设置页中的“专家 Agent 配置”区域在常见桌面窗口宽度下出现卡片横向挤压、badge 顶到标题、字段标签断裂和信息密度过高的问题，导致用户无法快速识别字段含义。

**目标**

在不修改配置读写逻辑、不影响 Windows/macOS 功能一致性的前提下，提升该区域的可读性、层级感和响应式表现，消除明显溢出与拥挤问题。

**非目标**

- 不调整后端或配置数据结构
- 不引入新的设置项
- 不重做整页视觉风格

**方案**

1. 将专家卡片改为更保守的响应式布局，默认单列，仅在超宽视口下使用双列，避免卡片被压缩过窄。
2. 将卡片头部改为纵向信息层级：角色名称/说明与权限摘要分离，badge 支持换行。
3. 为角色名、权限值、验证职责提供用户可读文案映射，减少直接暴露原始枚举值。
4. 将卡片内容分为“基础设置 / 权限设置 / 预算上限 / 风格提示”四个组，降低扫描成本。
5. 保持现有 `Input` / `Select` / `Switch` 组件与状态更新逻辑不变，只调整展示结构与类名。

**验证策略**

- 增加静态渲染测试，覆盖：可读标题、分组标题、可读权限摘要的渲染。
- 运行目标测试、全量 `npm test`、`npx tsc -p tsconfig.json --noEmit`。
- 如构建链路正常，再运行 `npm run build` 做一轮前端构建确认。
